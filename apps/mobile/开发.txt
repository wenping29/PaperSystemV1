# AI写作平台技术路线计划

## 上下文
基于README.md中的需求，需要开发一个AI写作平台，包含写作助手、评改评分、写作广场、好友功能、打赏功能等。用户要求技术路线简单，便于单人开发管理，需要拆分成客户端、服务端和后台管理平台，客户端支持安卓、苹果、PC Web端、平板和鸿蒙系统。

## 技术选型决策
根据用户选择：
- **客户端**：Flutter跨平台方案
- **服务端**：Python FastAPI单体应用
- **架构**：单体应用（初期简化）

## 推荐技术栈

### 客户端 (Flutter)
- **框架**：Flutter 3.x + Dart 3.x
- **状态管理**：Riverpod（推荐）或Bloc
- **网络请求**：Dio
- **本地存储**：SharedPreferences + Hive
- **推送通知**：Firebase Cloud Messaging（FCM）
- **支付集成**：官方插件或平台通道调用原生支付
- **鸿蒙适配**：初期使用WebView包装Flutter Web版本，或等待官方HarmonyOS NEXT支持
- **多平台输出**：
  - iOS：通过Xcode编译
  - Android：通过Android Studio编译
  - Web：Flutter Web编译为PWA
  - 桌面：Flutter桌面支持（Windows/macOS/Linux）
  - 鸿蒙：WebView或未来官方适配

### 服务端 (Python FastAPI)
- **框架**：FastAPI + Uvicorn
- **数据库ORM**：SQLAlchemy + Alembic（迁移）
- **数据验证**：Pydantic
- **认证授权**：JWT + OAuth2
- **AI集成**：OpenAI API、本地模型（通过transformers等）
- **实时功能**：WebSocket（FastAPI内置支持）
- **文件存储**：本地存储 + 云存储（阿里云OSS/腾讯云COS）
- **支付集成**：微信支付、支付宝官方SDK
- **缓存**：Redis（可选，初期可用内存缓存）

### 数据库
- **主数据库**：PostgreSQL（推荐15+）
- **缓存**：Redis 7+（可选，初期可用内存缓存）
- **文件存储**：MinIO（本地）或云存储服务

### 后台管理平台
- **技术栈**：React + TypeScript + Ant Design
- **与主服务关系**：同一FastAPI服务提供管理API，前端独立部署
- **认证**：使用同一JWT体系，区分管理员权限

## 数据库表结构（概要）

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE,
    password_hash VARCHAR(255),
    avatar_url TEXT,
    balance DECIMAL(10,2) DEFAULT 0.00,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 作品表
CREATE TABLE articles (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    is_public BOOLEAN DEFAULT true,
    view_count INT DEFAULT 0,
    like_count INT DEFAULT 0,
    reward_total DECIMAL(10,2) DEFAULT 0.00,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 评论表
CREATE TABLE comments (
    id UUID PRIMARY KEY,
    article_id UUID REFERENCES articles(id),
    user_id UUID REFERENCES users(id),
    content TEXT NOT NULL,
    parent_id UUID REFERENCES comments(id),
    created_at TIMESTAMP DEFAULT NOW()
);

-- 好友关系表
CREATE TABLE friendships (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    friend_id UUID REFERENCES users(id),
    status VARCHAR(20) DEFAULT 'pending', -- pending, accepted, blocked
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(user_id, friend_id)
);

-- 聊天消息表
CREATE TABLE messages (
    id UUID PRIMARY KEY,
    sender_id UUID REFERENCES users(id),
    receiver_id UUID REFERENCES users(id),
    content TEXT NOT NULL,
    is_read BOOLEAN DEFAULT false,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 打赏记录表
CREATE TABLE rewards (
    id UUID PRIMARY KEY,
    article_id UUID REFERENCES articles(id),
    from_user_id UUID REFERENCES users(id),
    to_user_id UUID REFERENCES users(id),
    amount DECIMAL(10,2) NOT NULL,
    message TEXT,
    is_anonymous BOOLEAN DEFAULT false,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 支付订单表
CREATE TABLE orders (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    order_no VARCHAR(50) UNIQUE NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) DEFAULT 'pending', -- pending, paid, refunded, failed
    payment_method VARCHAR(20), -- wechat, alipay
    payment_data JSONB, -- 支付平台返回数据
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

## API设计原则

1. RESTful风格，资源导向
2. 版本控制：`/api/v1/`
3. 统一响应格式：
   ```json
   {
     "code": 200,
     "message": "success",
     "data": {},
     "timestamp": "2026-03-31T10:30:00Z"
   }
   ```
4. 错误处理：HTTP状态码 + 错误信息
5. API文档：FastAPI自动生成Swagger UI

## 部署方案

### 开发环境
- Docker Compose包含：PostgreSQL、Redis、MinIO
- 本地运行FastAPI（热重载）
- 本地运行Flutter（热重载）
- 本地运行React开发服务器

### 生产环境
- 云服务器（阿里云ECS/腾讯云CVM）
- Nginx反向代理 + SSL证书
- Docker容器化部署
- 数据库云服务（RDS PostgreSQL）
- 对象存储（OSS/COS）
- CDN静态资源加速

### 持续集成
- GitHub Actions自动化测试
- Docker镜像构建
- 自动化部署脚本

## 单人开发管理建议

1. **版本控制**：Git分支策略（main/develop/feature）
2. **任务管理**：使用Trello/Notion记录功能点和进度
3. **代码质量**：配置pre-commit hooks（black, isort, flake8）
4. **测试策略**：关键路径单元测试 + 核心功能集成测试
5. **文档维护**：代码注释 + API文档 + 部署文档
6. **迭代节奏**：每2周一个可演示版本，收集反馈

## 风险与应对

1. **鸿蒙适配风险**：WebView方案体验可能不佳
   - 应对：密切关注Flutter官方对HarmonyOS的支持进展

2. **AI成本风险**：API调用费用可能较高
   - 应对：实现缓存机制，考虑本地模型部署

3. **支付安全风险**：资金交易需高度安全
   - 应对：使用官方SDK，严格验证回调，敏感信息加密

4. **内容审核风险**：用户生成内容需合规
   - 应对：实现关键词过滤，举报机制，人工审核后台

5. **性能扩展风险**：单体应用可能随用户量增长变慢
   - 应对：监控性能指标，预留微服务拆分可能性

## 总结

本计划采用Flutter + Python FastAPI + PostgreSQL技术栈，以单体应用起步，平衡了开发效率、维护成本和功能需求。适合单人开发管理，同时为未来扩展预留了空间。建议按照阶段逐步实施，每阶段都有可交付成果，持续迭代优化。
