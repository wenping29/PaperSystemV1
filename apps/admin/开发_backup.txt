 用户要求开发后台管理系统，用于管理AI写作平台的用户、内容、AI使用和商业功能。根据用户的选择，我们将优先实现用户管理
     功能，采用React + TypeScript + Ant Design作为前端技术栈，并遵循先完成后端API再开发前端的开发顺序。

     现有基础

     - 后端框架：FastAPI + SQLAlchemy + PostgreSQL
     - 权限系统：已存在USER, VIP, ADMIN角色和require_admin等权限检查函数
     - 用户管理API：已有基本的用户CRUD和管理员端点（在users.py中）
     - 前端规划：设计文档中指定了React + TypeScript + Ant Design技术栈

     实施策略

     第一阶段：后端API扩展（1-2周）

     优先扩展用户管理相关的管理员API，然后逐步添加其他管理功能。

     1.1 扩展用户管理API（在现有users.py中添加）

     目标：提供完整的管理员用户管理功能
     新增端点：
     - GET /admin/users/statistics - 用户统计（总数、按角色分布、活跃度等）
     - GET /admin/users/{user_id}/activities - 用户活动记录（登录、文章发布等）
     - POST /admin/users/{user_id}/ban - 封禁/解封用户
     - PUT /admin/users/{user_id}/balance - 调整用户余额
     - GET /admin/users/search/advanced - 高级搜索（按注册时间、最后登录、角色等）

     权限：所有端点使用Depends(require_admin)装饰器

     1.2 创建管理仪表板API（新建admin.py）

     目标：提供管理后台的概览数据
     新增端点：
     - GET /admin/dashboard - 仪表板数据（用户增长、文章增长、AI使用统计等）
     - GET /admin/system/health - 系统健康状态
     - POST /admin/operation-logs - 记录管理员操作日志

     1.3 创建缺失的数据库模型

     根据设计文档，创建以下模型：
     - Comment - 评论系统
     - Friendship - 好友关系
     - Message - 私信聊天
     - Reward - 打赏记录
     - Order - 支付订单

     注意：这些模型的基础CRUD API将在后续阶段添加。

     第二阶段：前端管理界面开发（2-3周）

     在apps/admin目录中初始化React项目，并实现用户管理模块。

     2.1 项目初始化

     npx create-react-app . --template typescript
     npm install antd @ant-design/icons axios react-router-dom dayjs
     npm install -D @types/react-router-dom

     2.2 核心架构

     - API客户端：创建统一的Axios实例，包含请求拦截器（添加JWT）和响应拦截器（处理401错误）
     - 路由配置：实现路由守卫，检查管理员权限
     - 状态管理：使用React Context或Zustand管理用户认证状态
     - 布局组件：创建AdminLayout（侧边栏 + 头部 + 内容区）

     2.3 用户管理模块

     - 用户列表页面：表格展示用户，支持搜索、筛选、分页
     - 用户详情页面：显示用户详细信息、活动记录
     - 用户操作：封禁/解封、角色修改、余额调整
     - 用户统计：图表展示用户增长、角色分布等

     第三阶段：其他管理功能扩展（1-2周）

     3.1 文章管理API和界面

     - 文章审核、状态修改、强制删除
     - 文章统计和数据分析

     3.2 AI使用管理API和界面

     - AI使用统计和报表
     - 用户配额管理
     - 使用记录查看

     3.3 评论、好友、消息管理

     - 实现基础的管理功能

     第四阶段：集成测试和部署（1周）

     - 权限安全测试

     验证方案

     后端API验证

     1. 使用FastAPI自动生成的Swagger UI测试所有管理员端点
     2. 验证权限控制：使用非管理员token访问应返回403
     3. 验证数据完整性：确保管理员操作不影响普通用户功能

     前端功能验证

     1. 登录流程：管理员登录成功，普通用户登录应重定向或显示无权限
     2. 用户管理功能：列表、搜索、筛选、操作（封禁、角色修改等）
     3. 权限控制：非管理员用户无法访问管理界面

     集成验证

     1. 前后端数据流：确保前端正确显示后端数据
     2. 错误处理：网络错误、权限错误、数据验证错误等
     3. 性能测试：页面加载速度、API响应时间

     部署考虑

     开发环境

     - 管理前端运行在3000端口，后端运行在8000端口
     - 配置CORS允许跨域请求
     - 使用环境变量管理API基础URL

     生产环境

     推荐同域部署方案：
     - Nginx配置反向代理，将/api/*转发到后端，其他请求到前端
     - 统一域名，避免CORS问题
     - 配置SSL证书

     后续扩展

     完成基础后台管理系统后，可以考虑：
     1. 实时数据监控（WebSocket）
     2. 数据导出功能（Excel, PDF）
     3. 多语言支持
     4. 自定义仪表板
     5. 审计日志和操作追溯

     风险与缓解

     1. 权限安全风险：确保所有管理员端点严格验证权限，记录操作日志
     2. 前端路由安全风险：实现路由守卫，检查JWT token中的角色
     3. API兼容性风险：新API使用/admin/前缀，与普通API分离
     4. 部署复杂性风险：采用同域部署，简化配置
